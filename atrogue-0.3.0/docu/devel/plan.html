Plans for Future Versions

Several interesting features have not yet been implemented. However, the current development focus shall not be on featuritis, but rather on fixing all bugs which appear and on correcting any game design mistakes before building more code on top of that, making it even worse.

This page only lists some very general/global feature ideas. Tons of little ideas are documented within the source code with "IMPLEMENTME" marks (cf. the coding style documentation).

  • Finetuning: pretty much anything in atrogue depends on chance; but some of the probabilities (and aging amounts BTW) surely need some finetuning, e.g. should be higher or lower or depend more (or less) on gameplay difficulty, storey number etc.
  • Shops: could implement special "shop sections" where the player can buy objects, paying with gold found during the game.
  • Other special sections: could implement "bonus/malus" sections where the player can pick enchanted objects or has to fight against nasty creatures or "structural issues" of the section (flood; shrinking room). Such sections could appear e.g. when the player applies the AoY.
  • Number of lives: the player character can already be reborn; but maybe some players would like to get more lives, e.g. using a command-line option; rather allow the player to increase the number with in-game activities, maybe a special scroll or potion or a special trick with a stone table obstacle or quarks.
  • Demo Mode: the demo mode shall allow the player to sit back and watch the game playing automatically. Minimal code is already there, but implementing good heuristics for an interesting automatic gameplay will be hard, and this shouldn't begin before all other relevant features are implemented - otherwise the heuristics become wrong.
  • Remote Mode: this is a rather crude idea about external programs which could start atrogue remotely, rerouting the standard input/output and faking a human player. Think of it as some kind of "external demo mode". Very low on my to-do list...
