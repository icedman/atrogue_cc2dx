Changes

Changes for version 0.3.0 (released 2010-02-22):

  • upgraded the license to GNU General Public License version 3 ("or newer")
  • overhauled the entire source code; changed many small details and some bigger ones; introduced the "section" concept and the "dungeon department" concept; renamed the source code files level.[ch] to dungeon.[ch]
  • introduced different roles in addition to "rogue", e.g. caveman, trader and wizard
  • implemented several kinds of objects and obstacles, e.g. keys and doors
  • implemented some keyboard commands, e.g. for door handling
  • implemented several kinds of creatures, special attacks and dungeon traps
  • added the documentation file docu/credits.html
  • implemented a "scrollable texts" feature, especially to make the user interface for inventory handling more flexible
  • implemented flexible lighting of rooms and individual squares

Changes for version 0.2.0 (released 2003-02-09; 172 KB):

  • split cfFlits into cfFlitsMov and cfFlitsAtt (creature.c, action.c)
  • implemented ackRefresh (action.c)
  • implemented cfPlantlike and fkVegetables (creature.c, object.c)
  • implemented rods (in addition to wands and staves, for diversity) (object.c)
  • implemented tokTowel - yum! (object.c, main.c)
  • moved code for the keyboard command "apply" from main.c to the new file apply.c to get smaller source code files (main.c, apply.c)
  • introduced support for configure scripts (for better portability etc.)

Changes for version 0.1.6 (released 2003-01-29; 161 KB):

  • let creatures step on (and glide over) sfGlazedFrost (action.c)
  • introduced the concept of "storeys", so that each storey can consist of more than one level/map in future versions (level.c)
  • wrote developer documentation about dungeon traps
  • implemented tokShawlGas, tokShawlFood, tokDie
  • let defeated creatures magically re-appear sometimes (by a delayed level event effect) (creature.c, level.c)

Changes for version 0.1.5 (released 2003-01-22; 156 KB):

  • improved the dungeon trap handling; implemented a "falling rocks" trap (level.c)
  • use finer-grained object weight calculations in order to avoid "rounding" problems (object.c)
  • implemented quarks, but no tea :-) (object.c, main.c)

Changes for version 0.1.4 (released 2003-01-15; 153 KB):

  • implemented mouse support (main.c)
  • use nice little object descriptions instead of plain pronouns in some messages (object.c)
  • implemented ccSpider creatures and shfSpins (creature.c)
  • implemented the keyboard command "glimpse" (main.c)
  • implemented a new object class "tools" (a generalization of the old class "lantern") (object.c)

Changes for version 0.1.3 (released 2003-01-08; 146 KB):

  • implemented rkSearch rings (object.c, main.c)
  • implemented ccArcher and ccWorm creatures (creature.c)
  • moved many general strings and helper functions from main.c to the new file stuff.c (main.c, stuff.c)
  • improved the "artificial intelligence" of creatures (action.c)
  • started the implementation of obstacles (generalization of chests) (level.c)
  • implemented a "pit" trap (level.c, main.c)

Changes for version 0.1.2 (released 2002-12-31; 141 KB):

  • implemented the "count/number" argument for several commands (main.c)
  • fixed the ordering of "put the player character into a level" and "populate a new level with creatures" (level.c)
  • overhauled the whole project

Changes for version 0.1.1 (released 2001-07-05; 141 KB):

  • started animation implementation (animation.c, level.c)
  • try to hoe walls resp. dig chests only for appropriate move-kinds (main.c)
  • fixed a little bug in map_can_player_see() (level.c)
  • put the tail of throw() into a new function creature_throw(); let creatures throw objects at the player character (main.c, creature.c, action.c)
  • implemented remains in creature_lose_all_objects() (creature.c)
  • implemented wkClub, changed wkMace attributes (object.c)
  • implemented the command-line option "--colorhl=..." for color correction (init.c)
  • increased the relative amount of wands/staves slightly and let more scrolls of identify appear (object.c)

Changes for version 0.1.0 (released 2001-06-25; 136 KB):

  • cleaned up special inventory handling for the player character (main.c, creature.c, level.c)
  • cleaned up aggressivity and cfHumanoid handling; allowed wear-armor and wield-weapon for any humanoid creature, not just for the player character (creature.c)
  • implemented the command "pick object" (main.c)
  • implemented wkHarpoon, wkBoomerang and wkSword; fine-tuned the weapon minlevels and damages (main.c, object.c)
  • let lantern description texts mention whether it's lit (object.c)
  • introduced mfWall and mfChest (no real change yet, only preparing for later improvements) (level.c, creature.c, main.c)
  • some little cleanup and fixes for identify() and do_cast_spell() (main.c)
  • cleaned up object hoe-value calculation further (main.c)
  • moved creature_act_all() and demo_act() into a new file "action.c", called them action_creatures() and action_rogue(), removed demo.[ch] and cleaned up that stuff (main.c, creature.c, demo.c, action.c)
  • improved the AI ("artificial intelligence") of creatures (action.c, level.c)
  • changed hit-chance calculations in creature_hit() - the game has become more difficult due to the improved AI... :-) (creature.c)
